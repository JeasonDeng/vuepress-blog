    # WWW

    万维网（World Wide Web）诞生于 1990 年，由 Tim Berners-Lee 发明 主要包含三个概念

    1. URI，俗称网址
    2. HTTP，两个电脑之间传输内容的协议
    3. HTML，超级文本，主要用来做页面跳转

    URI 的作用是能让你访问一个页面，HTTP 的作用是让你能下载这个页面，HTML 的作用是让你能看懂这个页面

    # URI是什么

    URI 分为 URL 和 URN，我们一般使用 URL 作为网址

    # URN

    通过 URN 你可以确定一个「唯一的」资源

    ISBN: 9787115275790 就是一个 URN，对应的资源的是《JavaScript 高级程序设计（第三版）》这本书

    # URL

    通过 URL 你可以确定一个「唯一的」地址（网址） URL 常见组成

    # DNS

    域名解析 我们输入网址后，电信会先对域名进行域名解析，拿到对应的 IP 地址，再去访问对应的服务器

    # 可获取当前域名对应的 ip 
nslookup baidu.com
# or 
ping baidu.com

    # 请求与响应

    Server + Client + HTTP

    • 浏览器负责发起请求
    • 服务器在 80 端口接收请求
    • 服务器负责返回内容（响应）
    • 浏览器负责下载响应内容

    HTTP 的作用就是指导浏览器和服务器如何进行沟通

    # 请求

    浏览器负责发起请求

    # 状态码

    状态码要背，是服务器对浏览器说的话

    • 1xx 不常用
    • 2xx 表示成功
      • 200 —— 请求已成功，请求所希望的响应头或数据体将随此响应返回。实际的响应将取决于所使用的请求方法。在 GET 请求中，响应将包含与请求的资源相对应的实体。在 POST 请求中，响应将包含描述或操作结果的实体
    • 3xx 表示滚吧，重定向
      • 301 —— 永久重定向，被请求的资源已永久移动到新位置
      • 302 —— 客户端临时重定向
    • 4xx 表示你丫错了，客户端出错
      • 403 —— 服务器已经理解请求，但拒绝执行它
      • 404 —— 所请求的资源在服务器上未被发现
    • 5xx 表示好吧，我错了，服务端出错
      • 500 —— 服务器出错，无法完成对请求的处理
      • 502 —— 作为网关或代理工作的服务器尝试执行请求时，从上游服务器接收到无效的响应

    # 用 Chrome 查看响应

    1. 打开 Network
    2. 输入网址
    3. 选中第一个响应
    4. 查看 Response Headers，点击「view source」
    5. 你会看到响应的前两部分
    6. 查看 Response 或者 Preview，你会看到响应的第 4 部分
