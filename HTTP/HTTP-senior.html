    # Cache-Control

    Cache-Control (opens new window)

    Cache-Control 通用消息头字段，被用于在http请求和响应中，通过指定指令来实现缓存机制

    // 后台代码
// max-age 后的数值单位是 s，表示从此时开始多少 s 后缓存失效，需要再次请求
// 在此时间段内同样的路径是不会重新请求的
res.setHeader('Cache-Control', 'max-age=3000000')

    # Cache-Control 与 ETag 有什么区别？

    Cache-Control 的 max-age = 1000

    • 表示加载了该缓存后，在 1000 秒内再次请求同一个 URL 时就不请求，直接使用上一次缓存

    ETag

    • 在请求后，此次请求头 If-None-Match 的值与 ETag 值一致，则不再进行加载新的请求文件，直接从缓存中拿。状态码为 304，每次都有请求，但相应是没有第四部分的，为空

    # Expries

    1. Expires 是以前用来控制缓存的http头，Cache-Control是新版的API。

    2. 现在首选 Cache-Control。

    3. 如果在Cache-Control响应头设置了 "max-age" 或者 "s-max-age" 指令，那么 Expires 头会被忽略。

    4. 响应头设置方式：

      // 第二个参数必须是格林威治时间
// var d = new Date()   d.toGMTString()
res.setHeader('Expires', 'Wed, 21 Oct 2015 07:28:00 GMT')
