    # 命令行是什么

    命令行就是终端黑窗口，用命令的形式来操作文件或编辑代码

    • Windows 的 cmd 或 Git Bash
    • Mac 的终端

    实际上是先有命令行，后有的图形界面；你可能以为命令行是异端，但其实图形界面才是异端

    # 常见的系统自带命令

    参考 (opens new window)

    命令 说明
    cd 目录路径 进入指定目录
    pwd 查看当前路径
    mkdir 目录名 创建文件夹
    mkdir -p 目录名/子目录名/子目录名 创建多级文件
    whoami 查看用户名
    - -
    ls 路径 列出指定路径下所有文件
    ls -a 路径 列出目录所有文件，包含以.开始的隐藏文件
    ls -l 路径 将文件的权限、所有者、文件大小等信息详细列出来
    - -
    echo '1' > 文件名 创建文件并写入 '1'
    echo '1' >! 文件名 强制写入 '1',覆盖原内容
    echo '1' >> 文件名 追加内容
    touch 文件名 创建文件
    - -
    cp 源路径 目标路径 复制文件
    cp -r 源路径 目标路径 复制文件夹
    - -
    mv 文件名 新文件名 重命名
    - -
    rm 文件路径 删除指定文件
    rm -f 文件路径 强制删除
    rm -r 目录路径 删除目录
    rm -rf 目录路径 强制删除目录
    - -
    curl -L https://www.baidu.com (opens new window) > baidu.html 下载文件
    wget -p -H -e robots=off https://www.baidu.com (opens new window) 拷贝网页资源 (Windows 不支持 wget)
    - -
    df -kh 查看磁盘占用情况
    du -sh 查看目录大小
    du -h 查看各文件大小

    # Vim

    如何退出 vim?

    1. 强制退出（不保存）—— 狂按 ESC，然后按下 :q! 回车
    2. 保存后退出 —— 狂按 ESC，然后按下 :wq 回车

    # .bashrc /.zshrc文件

    bashrc 和 zshrc 是一样的
    ~/.bashrc / ~/.zshrc 文件的功能很强大

    # 自动运行

    1. 首先 touch ~/.bashrc 创建一下这个文件
    2. start ~/.bashrc 选用编辑器编辑这个文件，内容为 echo 'Hi'
    3. 你也可以用命令行编辑文件 echo "echo 'hi'" >> ~/.bashrc
    4. 关闭退出 Git Bash，然后打开 Git Bash，是不是看到了 Hi
      • 这说明每次进入 Git Bash，就会优先运行 ~/.bashrc 里面的命令
    5. 重新编辑 ~/.bashrc，内容改为 cd ~/Desktop，重启 Git Bash，有没有发现默认就进入桌面目录了？

    你可以用 ~/.bashrc 在进入 Git Bash 前执行任何命令，十分方便

    # alias 别名

    1. 在 ~/.bashrc 里新增一行 alias f="echo 'frank is awesome'"
      • 等于号两边不能有空格，你最好一个字都不要错
    2. 运行 source ~/.bashrc，作用是执行 ~/.bashrc
    3. 运行 f，就会看到 frank is awesome
    4. 也就是说，现在 f 就是 echo 'frank is awesome' 的缩写了
      • 利用这个技巧，我们可以把很多常见的命令缩写一下，比如
    # ~/.bashrc
alias la='ls -a'

alias ll='ls -l'

alias gst='git status -sb'

alias ga='git add'

alias ga.='git add .'

alias gc='git commit'

alias gc.='git commit .'

    # 配置环境变量

    还可以在 ~/.bashrc 里面设置一些环境变量，比如你可以在 ~/.bashrc 里面添加一行

    export SASS_BINARY_SITE="https://npm.taobao.org/mirrors/node-sass"
    1

    那么以后你安装 node-sass 的时候就不会因为被墙而报错了。以后会用到的，现在先说一下

    # 设置 PATH

    export PATH="目录的绝对路径:$PATH"

    这句话可以在 PATH 里添加一个目录

    • 你每次在 Bash 里面输入一个命令时（比如 ls、cp、demo）
    • Bash 都会去 PATH 列表里面寻找对应的文件，如果找到了就执行
